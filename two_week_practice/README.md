# Two-Week Practice Reviews

Use these after every two-week block. They are meant for recall, pattern recognition, and confidence before moving forward.

How to use:

1. Do the review without looking at old solutions first.
2. If stuck, write the problem shape before writing code.
3. After solving, compare your approach with the pattern checklist.
4. Redo missed problems the next day from memory.

## Review Path

```text
two_week_practice/
├── weeks1_2_practice.md      # Python basics, decisions, loops, strings, lists
├── weeks3_4_5_practice.md    # functions, dictionaries, sets, mixed challenges
├── weeks6_7_practice.md      # DSA basics, arrays, searching, hash maps, stacks, queues
├── weeks8_9_practice.md      # recursion, linked lists, two pointers, trees, graphs
└── week10_capstone.md        # sorting, heaps, sliding window, final pattern review
```

## The Rule

Do not memorize code first. Memorize the question each pattern asks:

- What do I need to remember?
- Am I counting, searching, grouping, filtering, or tracking best so far?
- Do I need values, indexes, keys, or positions?
- Can I avoid repeating work?
